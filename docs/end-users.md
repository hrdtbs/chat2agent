# エンドユーザ向けガイド

この文書は **Slack や Discord でボットに依頼する人**向けです。組織でボット名やルールが違う場合は、社内の案内を優先してください。

## 何ができるか

ボットに **メンション**して、AI コーディングエージェント（例: Devin / Jules）への作業依頼を出せます。

- **セッション開始前**: システムが API 上必須な情報（例: Jules のリポジトリソースとブランチ）が足りないとき、**同じスレッド**で案内があり、返信で補足します。
- **セッション開始後**: Devin や Jules が追加で聞きたいことがあると、**同じスレッド**に質問が表示されることがあります。その内容に **返信**すると、エージェント側に伝わります。

## 始め方（共通）

1. ボットが入っているチャンネル（またはスレッド）を開く。
2. メッセージで **@ボット名** を付けて依頼内容を書く。
3. ボットから返信があったら、そのスレッドに **続けて返信** する（案内や追加質問に答える場合）。

## 例アプリ（`examples/nextjs`）のときの動き

このリポジトリ付属の例では、次のルールに近い挙動になります（本番でカスタムしている場合は異なります）。

### どちらのエージェントに送るか

- メッセージに `**jules`** という単語が含まれる（太字不要） → Jules 向けとして処理しようとします。
- メッセージに `**devin**` という単語が含まれる → Devin 向けとして処理しようとします。
- どちらもなければ、サーバ側の設定（環境変数 `CHAT2AGENT_DEFAULT_AGENT`、既定は `devin`）に従います。

### Jules を使うとき

Jules 用にルーティングされた場合、**Jules に登録済みのソース**と **ブランチ名** がセッション作成に必要です。例アプリでは、ボットが案内したとおりに **返信で** 次のような形を送ると解釈されることがあります。

- `sources/で始まるソース名`（1 行）
- ブランチ名（例: `main`）（別行）

管理者がサーバに `JULES_DEFAULT_SOURCE` などを設定している場合は、ソースの入力を省略できることがあります。

### Devin を使うとき

例アプリでは、メンション前に **リポジトリを `owner/repo` で書く必要はありません**。Devin がセッション中に必要なら、**スレッド上で質問**してくる想定です（組織のカスタム実装では異なる場合があります）。

### 起動前の案内が何度も続く場合（主に Jules）

Jules のソース/ブランチなど **API 必須項目**の収集には回数上限があります。上限に達した場合は、ボットが打ち切りメッセージを出すことがあります。そのときは **新しいメンション** で、案内どおりの情報をそろえて依頼し直してください。

## Slack と Discord の違い（操作感）

- **Slack**: チャンネルやスレッドで @メンション。返信は同じスレッドに投稿すると追いやすいです。
- **Discord**: サーバの運用ルールに従い、ボットが見ているチャンネルでメンションしてください。スレッド機能を使う場合は、そのスレッド内で会話を続けます。

## うまく動かないとき

次は **インテグレータ（管理者・開発者）** の対応範囲です。自分では直せない場合はその担当に伝えてください。

- ボットがチャンネルにいない / メンションできない
- 常にエラーだけ返る（API キーや権限の問題など）
- セッションは始まったが、追加質問が来ない（サーバ側のポーリングや Cron の設定など）
- 社内ポリシーで Devin / Jules の利用が制限されている

詳しい設定やトラブルシュートは [インテグレータ向けガイド](./integrators.md) を参照してください。
