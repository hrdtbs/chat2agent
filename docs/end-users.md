# エンドユーザ向けガイド

この文書は **Slack でボットに依頼する人**向けです。組織でボット名やルールが違う場合は、社内の案内を優先してください。

## 何ができるか

ボットに **メンション**して、**Devin** への作業依頼を出せます。

- **セッション開始前**: 組織の設定によっては、API 上必須な情報が足りないとき **同じスレッド** で案内があり、返信で補足します。
- **セッション開始後**: Devin が追加で聞きたいことがあると、**同じスレッド** に質問が表示されることがあります。その内容に **返信**すると、Devin 側に伝わります。

## 始め方

1. ボットが入っているチャンネル（またはスレッド）を開く。
2. メッセージで **@ボット名** を付けて依頼内容を書く。
3. ボットから返信があったら、そのスレッドに **続けて返信** する（案内や追加質問に答える場合）。

## 例アプリ（`examples/nextjs`）のときの動き

このリポジトリ付属の例では、メンションは **常に Devin** に送られます。メンション本文に `owner/repo` 形式の行を含めると、Devin の `repos` に渡す用途で解釈されることがあります（組織のカスタム実装では異なる場合があります）。

### 起動前の案内が何度も続く場合

**起動前**の必須情報の収集には回数上限があります。上限に達した場合は、ボットが打ち切りメッセージを出すことがあります。そのときは **新しいメンション** で、案内どおりの情報をそろえて依頼し直してください。

## うまく動かないとき

次は **インテグレータ（管理者・開発者）** の対応範囲です。自分では直せない場合はその担当に伝えてください。

- ボットがチャンネルにいない / メンションできない
- 常にエラーだけ返る（API キーや権限の問題など）
- セッションは始まったが、追加質問が来ない（サーバ側の `after()` ポーリングやホストの制限など）
- 社内ポリシーで Devin の利用が制限されている

詳しい設定やトラブルシュートは [インテグレータ向けガイド](./integrators.md) を参照してください。
